# Creative menu

Адаптивная навигация Creative на чистых HTML, CSS и JS.

## Структура проекта

- `index.html` - базовая разметка хедера, мобильной панели, callback-модалки и демо-контента.
- `styles.css` - входной CSS-файл, который подключает тематические файлы из папки `styles/`.
- `styles/base.css` - дизайн-переменные, базовые стили страницы.
- `styles/header.css` - шапка, логотип, desktop-меню, телефон, CTA и кнопки мобильного хедера.
- `styles/dropdown.css` - desktop dropdown, карточки модулей, соцсети и анимация появления элементов.
- `styles/modal.css` - затемнение, callback-модалка, поля, чекбокс, кнопка отправки и success-состояние.
- `styles/responsive.css` - брейкпойнты и мобильная навигация-аккордеон.
- `js/config.js` - пути к ассетам, брейкпойнты, тайминги анимаций.
- `js/media.js` - общие проверки media query для мобильной навигации и callback-модалки.
- `js/menu-data.js` - единый источник пунктов меню для desktop и mobile.
- `js/nav-render.js` - HTML-рендер desktop-меню, dropdown и мобильных аккордеонов.
- `js/navigation.js` - поведение desktop dropdown и мобильного аккордеона.
- `js/callback-modal.js` - открытие/закрытие модалки, focus trap, маска телефона, валидация и success-состояние.
- `js/main.js` - точка входа, общие обработчики `Escape`, resize и клика по затемнению.
- `assets/menu/` - логотип, иконки, изображения модулей и наград.

## Где менять меню

Главный источник данных находится в `js/menu-data.js` в массиве `navTree`.

Один и тот же `navTree` используется для desktop dropdown и мобильного меню-аккордеона. Если добавить, удалить или переименовать пункт в `navTree`, изменение автоматически попадет в обе версии.

Базовая структура пункта:

```js
{
  id: "development",
  label: "Разработка",
  href: "#",
  layout: "two-col",
  children: [],
  modules: [],
  awards: [],
  socials: []
}
```

Пункты без `children`, `modules` и специальных данных отображаются как обычные ссылки. Пункты с вложенными данными автоматически получают dropdown на desktop и раскрывающийся аккордеон на mobile.

## Где менять визуальные параметры

Основные цвета, радиусы, длительности анимаций и общие размеры вынесены в `styles/base.css`.

Перед добавлением нового цвета или тайминга сначала проверьте, нет ли подходящей переменной в `:root`.

## Брейкпойнты

- `1300px` - скрывается телефон в desktop-хедере.
- `1068px` - desktop-меню переходит на вторую строку.
- `694px` - включается мобильная навигация.
- `479px` - callback-модалка становится полноширинной.

JS-проверки брейкпойнтов находятся в `js/media.js`. Если меняется значение в `js/config.js`, проверьте связанные CSS-правила в `styles/responsive.css`.

## Проверка перед публикацией

- Открыть страницу на desktop и проверить hover dropdown для разделов с вложенностью.
- Проверить, что фокус по `Tab` проходит по интерактивным элементам хедера.
- Проверить мобильное меню на ширинах `320px`, `375px`, `480px`, `694px`.
- Проверить, что в мобильном меню одновременно открыт только один аккордеон.
- Проверить прокрутку длинных разделов, особенно `Магазин`.
- Проверить callback-модалку: открытие, закрытие, ошибки обязательных полей, success-состояние.
- Проверить focus trap callback-модалки: `Tab` и `Shift+Tab` должны циклически перемещать фокус только внутри модалки.
- Проверить, что в консоли браузера нет ошибок загрузки JS/CSS/ассетов.

## Важно

- Отправка callback-формы сейчас имитируется на фронтенде. Backend-интеграция намеренно не добавлялась.
- Не дублируйте меню вручную в HTML. Новые пункты нужно добавлять в `js/menu-data.js`.
- Если данные меню когда-нибудь будут приходить из CMS/API, оставьте экранирование через `escapeHTML()` в render-функциях.
